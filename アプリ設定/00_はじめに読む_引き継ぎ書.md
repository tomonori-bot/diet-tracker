## 1. このアプリは何か

**BodyLog** — パーソナルトレーナー向けの顧客管理アプリ（Webアプリ）。

- **思想**：トレーニングはトレーナーと顧客の二人三脚。会えるのは1日数時間だけ。
  「会っていない間」もモチベーションを落とさないようにする＝**継続支援ツール**。
- 詳しくは `10_思想とこだわり.md` を読む（機能の取捨選択に迷ったらここに戻る）。

### 主な機能（すべて本番で稼働中）
- トレーナーのアカウント登録・ログイン（アカウントごとに顧客データ分離）
- 顧客の管理（追加・編集・削除・検索・カルテ）
- 問診票（URL/QRで送信→記入→自動で顧客登録）
- セッション記録（種目×重量×回数×セット、マイ種目リスト）
- セルフトレーニングのチェックリスト＋14日継続率の可視化
- 顧客のスマホ記録（体重・モチベ・トレーニング・振り返り／1日複数回追記可）
- ビフォーアフター写真
- 体重・モチベのグラフ

## 3. 【最新の到達点】データ永続化が完了した（2026-06-09）

ここが直近で一番大きい変更。**必ず理解しておくこと。**

### 何が変わったか
- 以前：全データを `data/db.json` という1ファイルに保存 → Render再起動で**消えていた**
- 現在：**Supabase（外部DB）に保存** → 再起動・再デプロイでも**消えない**

### 仕組み（重要）
- データ層は `db.js` に集約されている
- `loadDB()` でSupabaseから全データを読み、`saveDB(db)` で**差分だけ**書き戻す
- **Supabaseには `@supabase/supabase-js` を使わず、REST API を fetch で直接叩いている**
  （理由：supabase-jsはNode20でWebSocketエラーになるため。RESTなら依存ゼロで安定）
- `SUPABASE_URL` / `SUPABASE_KEY` が未設定なら、自動で `data/db.json` ファイル保存にフォールバック
  （ローカル開発で便利）

### テーブル設計（ハイブリッドB）
Supabaseに3つのテーブル：
- `users`：トレーナーのアカウント（1人=1行、中身はJSON）
- `patients`：顧客（**1人=1行**、記録・セッション・写真もこの行のJSONに含む）
- `app_data`：intakeCodes / exercises / sessions(ログイントークン) をキー別に保管

> なぜこの設計か：顧客が増えても「1行ずつ」扱うので速い。かつ既存コードの大部分を流用できた。
> 将来もし「全顧客を横断して分析」したくなったら、その時 records 等を別テーブルに分ける（純粋B化）。
> 詳細は `20_実装計画_データ永続化.md`。

### 環境変数（Renderに設定済み）
| Key | 用途 |
|---|---|
| SUPABASE_URL | SupabaseのプロジェクトURL |
| SUPABASE_KEY | Supabaseの **service_role** キー（秘密。anonキーではない） |
| BACKUP_KEY | 全体バックアップAPI用の秘密キー |
| ADMIN_USER / ADMIN_PASS | デフォルト管理者（任意） |

> ⚠️ コードは環境変数の★や全角・余分なパス(/rest/v1)を自動で掃除する作りにしてある（db.js冒頭）。
>    過去にここでハマったので、その対策が入っている。

## 6. 開発のやり方（環境・手順）

### コードを変更したら
1. ローカルで `/Users/oohatatomonori/Downloads/diet-tracker-deploy` を編集
2. **必ずローカルで動作確認**（下記）してから
3. GitHub Desktop で commit → **Push origin**（＝Renderに自動デプロイ）

### ローカルでの動作確認方法（重要）
この環境にはNodeが標準で入っていない。検証時は一時的にNodeを使った：
```
# Node取得（一度きり。/tmp に展開）
# Apple Silicon用: node-v20.18.0-darwin-arm64
export PATH="/tmp/node-v20.18.0-darwin-arm64/bin:$PATH"

# 構文チェック
node --check server.js
node --check db.js

# ローカル起動（Supabaseに繋ぐ場合は環境変数をセットしてから）
export SUPABASE_URL="https://xxx.supabase.co"   # ★などを付けない！値だけ
export SUPABASE_KEY="eyJ...(service_roleキー)"   # eyJ で始まる長い文字列
PORT=3999 node server.js
# → [db] Supabase mode (REST) と出ればOK
# 環境変数なしで起動すると [db] File mode（ローカルのdb.jsonに保存）になる
```

### Push時の注意
- コマンドラインからの `git push` は認証が通らない → **GitHub Desktopでpushする**
- 翻訳拡張がSupabase管理画面を壊すことがある → Supat/操作はシークレットウィンドウ推奨
