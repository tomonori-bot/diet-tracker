# BodyLog — 身体記録管理アプリ

患者さんが毎日の体重・モチベ・運動量を記録 → スタッフ管理画面にリアルタイム反映 → グラフで推移管理。
**PC・スマホどちらからでもログインでき、同じデータをリアルタイム共有**します。

## 主な機能

### ✅ 患者画面 (`/patient.html?id=<患者ID>`)
- スマホ最適化された日々の記録フォーム（体重・モチベーション・運動量・メモ）
- 送信ボタン1回でサーバーへ転送、**管理画面の対象患者データに自動保存・反映**
- オフライン時は IndexedDB に保存 → オンライン復帰時に自動送信
- PWA対応（ホーム画面追加可能）

### ✅ 管理画面 (`/admin.html`)
- **ログイン認証**（PC・スマホで共通アカウント、トークンは30日有効）
- 患者一覧／患者追加・編集・削除
- 患者ごとの記録（体重・痛み・姿勢・モチベ・運動量）の閲覧／編集／追加
- セッション記録（施術ノート）
- **体重推移グラフ**（Chart.js）／**痛み・姿勢・モチベグラフ**
- **SSE (Server-Sent Events) によるリアルタイム通知** — 患者が記録した瞬間に管理画面のグラフ・一覧が更新
- 通知許可があればブラウザ通知も発火

### ✅ デバイス間共有
- データは全てサーバー側 (`data/db.json`) に保存
- PCで作成 → スマホでログイン → 同じデータを確認・編集可能
- 30日有効のトークンセッション（「ログイン状態を保持」ON時）

## 起動

```bash
npm install
node server.js
# http://localhost:3000 にアクセス
```

### 環境変数
| 変数         | デフォルト       | 説明                         |
| ------------ | --------------- | ---------------------------- |
| `PORT`       | `3000`          | リッスンポート               |
| `ADMIN_USER` | `admin`         | 管理者ユーザー名             |
| `ADMIN_PASS` | `bodylog2025`   | 管理者パスワード             |

> 本番運用時は **必ず `ADMIN_PASS` を変更してください**。

## ログイン情報（デフォルト）

| 項目         | 値             |
| ------------ | -------------- |
| ユーザー名   | `admin`        |
| パスワード   | `bodylog2025`  |

## データフロー

```
┌──────────────┐  POST /api/patients/:id/self-record   ┌─────────────┐
│  患者画面    │ ──────────────────────────────────▶  │   サーバー  │
│ (patient.html)│                                       │ (server.js) │
└──────────────┘                                       └──────┬──────┘
                                                              │ writeDB
                                                              ▼
                                                       data/db.json
                                                       (patientRecords + records)
                                                              │
                              SSE 通知                        │
┌──────────────┐  ◀────────────────────────────────────────── ┘
│  管理画面    │     EventSource /api/patients/:id/stream
│ (admin.html) │     → グラフ／一覧を即時再描画
└──────────────┘
```

- 患者画面で送信 → サーバーが `patientRecords` と `records`（自動インポート済）両方に保存
- SSEで管理画面に即時通知 → グラフ・記録一覧・ダッシュボードが自動更新
- フォールバック: SSEが切れた場合は8秒間隔ポーリングに自動切替
